# Zomato Restaurant Rating Prediction

An end-to-end machine learning project that analyzes the Zomato restaurants dataset, builds a model to predict a restaurant's rating class (Poor / Average / Good / Very Good / Excellent), and serves predictions through a Streamlit web app.

## Project Overview

This project has two parts:

1. **Exploratory Data Analysis & Model Building** (`Zomato_dataset_EDA.ipynb`)
   - Loads and merges the Zomato dataset with country codes
   - Explores missing values, duplicates, and outliers
   - Visualizes rating distributions, top countries, top cuisines, and cost by city
   - Encodes categorical features and scales numerical features
   - Trains and compares several regression models (Linear Regression, SVR, Decision Tree, Random Forest, KNN, AdaBoost) using `GridSearchCV`
   - Saves the best-performing model (`RandomForestRegressor`) and the fitted `StandardScaler` with `joblib`

2. **Prediction App** (`app.py`)
   - A Streamlit app where a user enters basic restaurant details (average cost for two, table booking, online delivery, price range) and gets back a predicted rating category

## Files

| File | Description |
|---|---|
| `Zomato_dataset_EDA.ipynb` | Full EDA, data cleaning, feature engineering, and model training notebook |
| `app.py` | Streamlit app for making predictions |
| `mlmodel.pkl` | Trained model, saved via `joblib` (generated by the notebook) |
| `scaler.pkl` | Fitted `StandardScaler`, saved via `joblib` (generated by the notebook) |
| `Zomato Encoded Dataset.csv` | Cleaned/encoded dataset used to fit the scaler as a fallback (generated by the notebook) |

> **Note:** `mlmodel.pkl`, `scaler.pkl`, and the dataset CSVs are not included in this repo by default — you need to run the notebook first to generate them (see below).

## Setup

### 1. Clone/download the project and install dependencies

```bash
pip install streamlit numpy pandas scikit-learn joblib matplotlib seaborn openpyxl
```

### 2. Get the data

You'll need the original Zomato dataset (`zomato.csv`) and the country code mapping (`Country-Code.xlsx`) in the same folder as the notebook.

### 3. Run the notebook

Open and run `Zomato_dataset_EDA.ipynb` top to bottom. This will:
- Produce intermediate CSVs (merged, outlier-treated, encoded datasets)
- Train the models
- Save `mlmodel.pkl` and `scaler.pkl` in the project directory

### 4. Run the app

Make sure `mlmodel.pkl`, `scaler.pkl`, and `Zomato Encoded Dataset.csv` are in the same directory as `app.py`, then run:

```bash
streamlit run app.py
```

The app will open in your browser (default: `http://localhost:8501`).

## How the App Works

The app takes four inputs:
- **Average cost for two** — numeric input
- **Table booking available?** — Yes/No
- **Online delivery available?** — Yes/No
- **Price range** — 1 (cheapest) to 4 (most expensive)

These are scaled using the saved `StandardScaler` and passed to the trained Random Forest model, which predicts an aggregate rating. The predicted numeric rating is then mapped to a category:

| Predicted Rating | Category |
|---|---|
| < 2.5 | Poor |
| 2.5 – 3.49 | Average |
| 3.5 – 3.99 | Good |
| 4.0 – 4.49 | Very Good |
| ≥ 4.5 | Excellent |

## Model

The final model used in the app is a **Random Forest Regressor**, tuned with `GridSearchCV` over `max_depth` and `n_estimators`, selected after comparing it against Linear Regression, SVR, Decision Tree, KNN, and AdaBoost using MAE and RMSE.

Features used for prediction:
- `Average Cost for two`
- `Has Table booking`
- `Has Online delivery`
- `Price range`

## Known Issues / Improvement Ideas

- `st.set_page_config()` is called after `st.title()` in `app.py` — Streamlit requires `set_page_config` to be the very first Streamlit command in the script, so this line should be moved to the top or it may raise an error.
- Only 4 of the many available features are used for prediction; incorporating more (e.g. cuisines, city, votes) could improve accuracy.
- The fallback scaler-fitting logic in `app.py` re-fits on the whole encoded dataset if `scaler.pkl` isn't found or invalid — keep the encoded CSV in sync with the notebook to avoid mismatched scaling.
